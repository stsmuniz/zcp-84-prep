# Mock / Simulados – ZCP 8.4

Esta pasta concentra mini-simulados e simulados completos usados na preparação para a certificação **Zend PHP Certified Engineer (PHP 8.4)**.

A ideia é:

- Guardar aqui todos os simulados em arquivos individuais (`mini-simulado-XX.md`, `simulado-completo-XX.md`, etc.).
- Registrar no quadro abaixo:
  - data em que foi feito,
  - tipo (mini / completo),
  - foco principal (tópicos),
  - percentual de acerto,
  - link para o arquivo do simulado,
  - link para as notas/análises em `notes/` (quando houver).

## Como usar esta pasta

1. **Criar o simulado**

   - Adicionar um novo arquivo `.md` em `mini-simulados/` ou `full-simulados/`.
   - Incluir enunciados, alternativas e, ao final, o gabarito (se preferir, separar o gabarito em outro arquivo para poder refazer sem olhar).

2. **Aplicar o simulado**

   - Fazer as questões em tempo real, simulando ambiente de prova (sem consultar documentação).
   - Registrar as respostas e calcular o percentual de acerto.

3. **Registrar o resultado**

   - Atualizar a tabela acima com:
     - número do simulado,
     - data,
     - tipo,
     - foco (ex: “Arrays e Strings”, “OOP/SPL/Error Handling”),
     - percentual de acerto,
     - links para o arquivo do simulado e para as notas.

4. **Analisar erros**
   - Criar/atualizar um arquivo em `notes/` (ex: `notes/2026-01-10-simulado-02-analise.md`) com:
     - lista de questões erradas,
     - por que errou,
     - qual conceito precisa ser reforçado,
     - links para snippets relevantes (se existirem).

---

## Objetivo de performance

Meta sugerida antes de comprar o voucher da prova:

- Atingir **pelo menos 85–90% de acerto** em:
  - 2–3 mini-simulados recentes (tópicos variados),
  - 1–2 simulados completos misturando todos os temas do exame.

Quando essas metas estiverem sendo atingidas de forma consistente (sem “chutar” excessivamente), é um bom indicativo de prontidão para a prova.
